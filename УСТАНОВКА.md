# Установка и запуск

Этот файл нужен студенту, который хочет поставить skills в Claude Code, Codex или другой код-агент с поддержкой папок `skills`.

## 1. Положить skills в агент

Скопируйте четыре папки из `skills/`:

```text
skills/content-pipeline/
skills/carousel-builder/
skills/reels-scriptwriter/
skills/email-sequence-writer/
```

Пример структуры после установки:

```text
.claude/skills/content-pipeline/SKILL.md
.claude/skills/carousel-builder/SKILL.md
.claude/skills/reels-scriptwriter/SKILL.md
.claude/skills/email-sequence-writer/SKILL.md
```

Для Codex обычно используется:

```text
~/.codex/skills/content-pipeline/SKILL.md
~/.codex/skills/carousel-builder/SKILL.md
~/.codex/skills/reels-scriptwriter/SKILL.md
~/.codex/skills/email-sequence-writer/SKILL.md
```

Готовая команда агенту:

```text
Скопируй папки content-pipeline, carousel-builder, reels-scriptwriter и email-sequence-writer из этого архива в папку skills того агента, в котором я работаю. Потом покажи итоговое дерево файлов и проверь, что каждый SKILL.md читается.
```

## 2. Проверить установку

```text
Покажи, какие skills доступны в этом проекте. В списке должны быть content-pipeline, carousel-builder, reels-scriptwriter и email-sequence-writer.
```

## 3. Запустить сборку из одного источника

Источник может быть любым:

- расшифровка видео;
- запись эфира или вебинара;
- статья;
- лендинг;
- заметки эксперта;
- отзывы клиентов;
- исследование аудитории;
- конкурентный пост, если задача не копировать, а понять рынок.

Базовый промт:

```text
Возьми источник из папки [путь] или из файла [путь]. Сначала собери source brief: core thesis, claim ledger, proof ledger, VOC ledger, forbidden assumptions, freshness check, angle map и format fit. Затем предложи 3-5 тем недели. Ничего не выдумывай. Где данных не хватает, ставь placeholder и помечай, что уточнить.
```

## 4. Выбрать тему и собрать пакет

```text
Выбираю тему [тема]. Запусти content-pipeline и собери пакет недели: Instagram post, X или Threads thread, Telegram post, Reels/TikTok/Shorts script, carousel brief, email. Перед финальной генерацией покажи стоп-гейт: POV, Hook Lab winners, форматный план и риски.
```

## 5. Развернуть форматы

```text
План утверждаю. Разверни форматы в папку проекта: 00-source/source-brief.md, 10-package-week/01-post.md, 02-thread.md, 03-carousel.md, 04-reels.md, 05-telegram.md, 06-email.md, queue.md и qa-report.md.
```

## 6. Картинки и карусель

Для карусели:

```text
Карусель собирай через carousel-builder. Сначала выбери archetype и покажи slide plan. Картинки не генерируй, пока я не утвержу текст и сцены. Если image-движок не подключен, выдай тексты слайдов, сцены и промты.
```

Для обычных обложек:

```text
Собери промты под картинки для каждого формата. Перед генерацией прочитай engine passport и покажи, какой движок выбран, какие размеры нужны и какие ограничения есть.
```

## 7. Что остается руками

- Утверждение темы и углов.
- Проверка фактов, обещаний, кейсов, цифр и юридических формулировок.
- Подключение image-движка, если нужна генерация картинок.
- Публикация в соцсетях и отправка email.
- Обновление `freshness-sources-2026.md` и `06-engine-passport.md`, если правила платформ или модели поменялись.
