# Freshness Sources 2026

Этот файл не является вечным справочником. Он нужен как список мест, которые нужно проверить перед записью урока, запуском курса или продакшеном клиента.

Verified at: 2026-06-21.

## Instagram / Reels

Official references:

- https://creators.instagram.com/blog/the-latest-with-instagram
- https://creators.instagram.com/faq

Что проверять:

- eligibility and recommendation limits for Reels;
- safe areas and UI overlays;
- current creative tools;
- changes in recommendations guidance.

Практическое правило для skills:

- не обещать алгоритмический результат;
- держать первые секунды сильными;
- делать 9:16 visual-first сценарии;
- использовать 7-12 sec, 20-45 sec и 60-180 sec как рабочие режимы, если источник держит внимание.

## TikTok

Official reference:

- https://ads.tiktok.com/business/en/creative-codes

Что проверять:

- Creative Codes;
- hook/body/close;
- TikTok-first creative;
- trend context;
- sound and production basics;
- ad and organic creative differences.

Практическое правило для skills:

- видео не должно звучать как рекламный диктор;
- visual, sound and text overlay должны работать вместе;
- тренд использовать только если он усиливает смысл.

## X

Official references:

- https://help.x.com/en/using-x/types-of-posts
- https://help.x.com/en/using-x/x-premium

Что проверять:

- standard post limits;
- long post limits for Premium;
- media and reply behavior;
- changes in link previews and visibility.

Практическое правило для skills:

- короткий пост не заменять лонгридом;
- long post использовать только если первые строки держат самостоятельный интерес;
- thread делать только если мысль реально требует цепочки.

## Gmail sender guidelines

Official references:

- https://support.google.com/mail/answer/81126
- https://blog.google/products/gmail/gmail-security-authentication-spam-protection/

Что проверять:

- authentication requirements;
- bulk sender thresholds;
- one-click unsubscribe;
- spam complaint thresholds;
- From domain alignment;
- sender compliance changes.

Практическое правило для skills:

- письма писать plain-text-first;
- один главный CTA;
- не перегружать ссылками;
- не имитировать личное письмо, если это массовая промо-рассылка;
- добавлять compliance reminder в metadata.

## OpenAI / image generation / Codex

Official references:

- https://help.openai.com/en/articles/6825453-chatgpt-release-notes
- https://platform.openai.com/docs

Что проверять:

- current image model names;
- edit/generation capabilities;
- supported aspect ratios;
- availability in ChatGPT, API, Codex or external tools;
- limits and pricing.

Практическое правило для skills:

- не прошивать конкретное имя модели как вечную истину;
- хранить текущие возможности в `skills/carousel-builder/references/06-engine-passport.md`;
- если данные старше 30 дней, обновить перед генерацией.

## Rule for agents

Если задача зависит от актуальных правил платформы, цен, моделей, лимитов, deliverability или законов, агент обязан:

1. проверить свежий источник, если доступен интернет;
2. иначе пометить факт как `needs current verification`;
3. не строить обещание на непроверенном факте.
