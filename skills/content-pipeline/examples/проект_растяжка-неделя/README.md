# Живой прогон сборщика — «растяжка», неделя из одного материала

> Это реальный прогон скилла `content-pipeline` (демо для урока М7-05). Источник — НЕ выдуман: папка контент-контура прошлого модуля, ниша «онлайн-студия растяжки» (`m6_content_summary.md` + `m6_content_queue.md`). Одна тема развёрнута в неделю контента под все площадки.

## Что сделал сборщик (по шагам урока)

**Шаг 1 — источник → тема.** Прочитал разбор аудитории, вытащил боли, предложил 5 тем, выбрана одна: **«Ты не деревянный — у тебя зажаты не те мышцы»** (стартовый страх новичка «я деревянный»). Подробно — `00-источник/тема-и-выбор.md`.

**Шаг 2 — нашёл готовые кубики.** Под карусель / Reels / письмо вызвал готовых ассистентов, не писал заново:
- карусель → `carousel-builder` (Урок 1)
- Reels → `reels_scriptwriter` (прошлый модуль, ниша переопределена на растяжку)
- письмо → `email-sequence-writer` (прошлый модуль)
- пост / тред / Telegram → готовых нет, собрал сам по `references/posts-threads-telegram.md`.

**Шаг 3 — план пакета → Шаг 4 — развернул в форматы.** 6 единиц, каждая под свою площадку, один голос бренда:

| Формат | Файл | Кем собран |
|---|---|---|
| Пост (лента) | `10-пакет-неделя/01-post.md` | сборщик (своя инструкция) |
| Тред (X/Threads) | `10-пакет-неделя/02-thread.md` | сборщик (своя инструкция) |
| Карусель (7 слайдов) | `10-пакет-неделя/03-carousel-каркас.md` | вызов `carousel-builder` |
| Reels / TikTok | `10-пакет-неделя/04-reels.md` | вызов `reels_scriptwriter` (★ reuse) |
| Telegram (канал) | `10-пакет-неделя/05-telegram.md` | сборщик (своя инструкция) |
| Письмо (value) | `10-пакет-неделя/06-email.md` | вызов `email-sequence-writer` |

**Шаг 5 — картинки + раскладка.** Промты картинок под размеры площадок + объём/цена (карусель пропущена) — `10-пакет-неделя/images/промты-картинок.md`. Раскладка по датам со статусом «черновик» — `queue.md`.

**Проверка чужими глазами** (4 фишки автора) — `проверка-чужими-глазами.md`.

## Честная граница прогона
- Тексты, промты, очередь — собраны end-to-end на реальном источнике (этот прогон).
- **Рендер картинок и публикация — руками** (движок gpt-image-2 / звено публикатора): сборщик отдаёт готовые промты + объём/цену и раскладку, кнопку «сгенерировать» и «опубликовать» нажимает человек.
- Картинки с лицом (пост, Reels) требуют фото-референса автора — на ученике.
