# Режим «текст кодом» (вёрстка → PNG) — опция, сильна в Claude Code

> Когда брать: нужна идеально точная вёрстка, лёгкая правка текста одной фразой, гарантированно единый фирменный шрифт на всех слайдах. Силён в **Claude Code** (сам ставит рендер и собирает PNG). В **Codex** обычно проще встроенный генератор (текст в картинке, режим A) — но код-вариант показать можно как «и так тоже можно».
>
> Это НЕ замена основному режиму A (текст в картинке), а опция рядом с ним. Дефолт — A.
>
> 🟠 **Статус источника.** Сам код-режим и подбор шрифтов — НАШЕ (практики рендера + подбор), Vibe рисует текст моделью, не кодом. В скилле честно помечать: это наш путь, не боевой Vibe.

## Идея

Картинка-фон генерится **без текста** (чистая сцена по стилю и плану директора), а текст слайда **верстается HTML/CSS и накладывается кодом** → рендер в PNG. Текст становится настоящим текстом: правится одной фразой, кириллица идеальна, шрифт и бренд едины на всех слайдах.

## Что реально умеет код-режим (чего image-модель не даёт)

Закладывая вёрстку, помни — здесь ты получаешь то, что модель в кадре не гарантирует:

1. **Текст = настоящий текст.** Правится одной фразой без перегенерации картинки. Опечатку, цену, имя поправил — пересобрал за секунду.
2. **Один фирменный шрифт пиксель-в-пиксель на ВСЕХ слайдах.** Брендовая консистентность, которую image-модель держать не обязана (она рисует *похоже*, см. `06-модели-возможности.md`).
3. **Снайперская вёрстка.** Точные отступы, safe-зоны от краёв, выравнивание, межстрочный, перенос строк — миллиметрово, не «как модель решила».
4. **Кириллица идеальна** — никаких кривых букв и артефактов в русском тексте.
5. **Эффекты текста через CSS:** тень, обводка (контур), градиент-заливка букв, полупрозрачная плашка-подложка, blur-стекло. Всё контролируемо.
6. **Реальное фото лица/товара вставкой** `<img>` прямо в вёрстку — 100% сходство (это настоящее фото, а не генерация).
7. **Бесплатно и приватно** — рендер локальный, ничего не уходит в облако.

**Честная граница (чего код НЕ даёт):** сам арт/сцену он не рисует — фон всё равно от image-модели (или сток/градиент), код кладёт текст ПОВЕРХ. Эффект «буквы вплавлены в сцену, за объектом, по форме предмета» — это режим A (image-модель), а не CSS.

## Шаги (агент делает сам, руками ученик ничего не пишет)

1. **Фон.** Тот же промт картинки (`02-styles.md` + `03-director.md`), но **БЕЗ секции TEXT** — добавить `background scene only, NO text, no letters, no captions on the image`.
2. **Вёрстка слайда.** HTML/CSS: фон-картинка как `background` + текстовый слой (`headline`/`subtext` из `01-text-phase.md`), шрифт и цвета из бренд-файла, safe-зоны (отступы от краёв). Заголовок крупно сверху, подпись ниже.
3. **Рендер.** Headless-браузер (Playwright) скриншотит вёрстку в PNG **1080×1350**. Фолбэк — Pillow (Python): текст поверх фона.
4. **Лицо/товар.** Либо в фон-картинке (identity-lock из `04-image-prompt.md`), либо реальное фото вставкой прямо в вёрстку (`<img>` в HTML).

## Кириллические шрифты (наш подбор — все с поддержкой кириллицы, бесплатные Google Fonts)

Image-модель шрифт не выбирает по имени — а здесь выбираешь точно. Проверенный набор под карусели (у всех есть кириллица):

**Для заголовков (display / bold — мощные, цепляют):**
- **Montserrat** — геометрический гротеск, классика сильных заголовков.
- **Unbounded** — яркий дисплейный, для смелых обложек (проверить набор начертаний под задачу).
- **Onest** — современный нейтральный, хорош и в заголовке, и в тексте.
- **Golos Text** — чистый российский гротеск, отличная кириллица.

**Для подписей / тела (читаемые):**
- **Inter** — эталон читаемости, идеальная кириллица, нейтральный.
- **Manrope** — современный, мягче Inter.
- **PT Sans** / **PT Root UI** — надёжная кириллица, спокойный тон.

**Для serif-стиля (LIFESTYLE из `02-styles.md`):**
- **Playfair Display** — высококонтрастный serif для эстетичных заголовков (кириллица есть).
- **PT Serif** — спокойный книжный serif с кириллицей.

Пара по умолчанию, если бренд-файла нет: **Montserrat (заголовок) + Inter (тело)** — беспроигрышно и строго по теме.

## Как загрузить шрифт (3 способа, по среде)

**A. Google Fonts по ссылке (`<link>`) — проще всего для Playwright.**
В `<head>` вёрстки:
```html
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@800&family=Inter:wght@400;600&display=swap" rel="stylesheet">
```
⚠️ **Критичный нюанс Playwright:** перед скриншотом дождаться, пока шрифт реально загрузится — иначе кадр отрисуется системным шрифтом. В коде рендера:
```js
await page.evaluate(() => document.fonts.ready);   // ждём загрузки всех шрифтов
await page.screenshot({ path: 'slide-01.png' });
```

**B. Локально (`@font-face`) — для оффлайна и гарантии.**
Скачать `.woff2`/`.ttf` (с fonts.google.com или через npm `@fontsource/montserrat`), положить рядом и подключить:
```css
@font-face { font-family: 'Montserrat'; src: url('./fonts/Montserrat-ExtraBold.woff2') format('woff2'); font-weight: 800; }
```
Плюс — рендер не зависит от сети, шрифт точно тот же на всех слайдах.

**C. Для Pillow (Python-фолбэк) — нужен `.ttf`/`.otf` локально:**
```python
from PIL import ImageFont
font = ImageFont.truetype("fonts/Montserrat-ExtraBold.ttf", 64)
```
Google Fonts отдаёт `.ttf` — скачать файл начертания и указать путь. (Системные шрифты Windows — `C:\Windows\Fonts\`.)

## Среда

- **Claude Code** — основной носитель режима: ставит Playwright/Pillow сам одной командой, рендерит локально и приватно. Бесплатно.
- **Codex** — встроенный генератор рисует сразу с текстом (режим A) и обычно удобнее; код-вёрстку показываем как «и так можно», без углубления.

## Что остаётся от Vibe

Углы темы, копирайт, стили, директор сцен, бренд-единство — те же (`references/01-03`). Меняется только финал: текст не внутри картинки, а кодом поверх чистого фона. Стиль-якорь и identity-lock работают так же (фон держит единый вид, лицо — в фоне или реальным фото в вёрстке).
