# Паспорт движков — что реально умеет модель (читать ПЕРЕД генерацией)

> **Зачем этот файл.** Image-модель — мощный инструмент, но в длинном промте агент про это «забывает» и просит у неё уровень обычной картинки. Здесь зафиксировано, что движок реально может, чтобы Дизайнер закладывал в промт ВЕСЬ потенциал: дизайнерский слайд с текстом внутри кадра, лицо и товар точь-в-точь, нужное начертание, правильный размер, правка готового кадра.
>
> Источник возможностей — боевой toolbox Виктора: `~/.claude/skills/инструменты/` (`codex.md`, `openai.md`, `replicate.md`). Это рабочие вызовы, не память.
>
> ⚠️ **Цифры стареют.** Версии, размеры, лимиты фото, цены, даты выхода меняются. Перед записью урока / важной генерацией сверять с доками toolbox и офиц. страницами (gpt-image-2 — developers.openai.com; nano-banana-pro — Replicate `google/nano-banana-pro`). В тексте для ученика меняющиеся числа не зашивать намертво — помечать «проверить на день записи».

## Движок A — gpt-image-2 (через Codex CLI, ДЕФОЛТ)

Как зовём: `bash /c/AI-projects/bin/codex-img.sh -p "PROMPT" -o out.png` (бесплатно по подписке ChatGPT, $0 в API). Подробно — `инструменты/codex.md`.

**Что реально умеет (закладывай в промт):**
- **Текст ВНУТРИ кадра, включая кириллицу.** gpt-image-2 хорошо рендерит русский/украинский текст прямо на картинке — это основа Варианта Б (дизайнерский слайд целиком). Поддерживает и латиницу, и иероглифы. → проси заголовок+подпись прямо в кадре (секция TEXT из `04-image-prompt.md`).
- **Reasoning перед генерацией + до 2K.** Модель «думает» над композицией до отрисовки, поэтому держит сложную сцену со связью текст↔визуал. `quality: high` = максимум деталей.
- **Размеры (флаг `-s`):** `1024x1024` (квадрат) · `1536x1024` (ландшафт/обложка) · `1024x1536` (вертикаль) · `auto`.
  ⚠️ **Точный формат ленты.** Вертикаль Codex = `1024×1536` — это пропорция **2:3**, а лента Instagram = **1080×1350 (4:5)**, она ниже. Codex ровно 4:5 не выдаёт (в флаг `-s` его не подать). Что делать: для сторис/теста `1024x1536` годится (сторис даже ближе к 9:16); нужен ровно 4:5 ленты — генери через **Replicate nano-banana-pro** с `aspect_ratio: "4:5"` (Движок B) или кропни `1024×1536` до `1024×1280`.
- **Фото-референсы (флаг `-i`, повторять для нескольких):** лицо, товар, style-anchor. Так держится identity-lock и единый бренд. **Style-anchor на практике:** сгенерь cover первым, затем каждый следующий слайд — с готовым cover как референсом:
  ```bash
  bash /c/AI-projects/bin/codex-img.sh -p "ПРОМТ слайда 2" -i "<папка>/slide-01.png" -o "<папка>/slide-02.png" -s 1024x1536 -q high
  ```
  Есть фото лица/товара — добавь их тоже отдельными `-i` (порядок: cover-anchor → лицо/товар).
- **Мульти-картинка консистентность** — серия в одном визуальном языке (опора style-anchor'а).

**Чего НЕ умеет встроенный Codex-генератор:**
- **Edit файла с диска** («возьми этот PNG и поменяй фон») — встроенный image_gen не правит локальный файл. Для правки готового кадра → gpt-image-2 через OpenAI API (`openai.md`, `images.edit`, принимает несколько референсов) или Flux Kontext через Replicate.
- **Видео** — не умеет (это другие движки, не для карусели).

**Если Codex капризничает — это норма, не пугайся:**
- Первый вызов может упасть на «холодном» старте (`exit 2` / `exit 127` — не доинициализировался). **Повтори один раз** — обычно идёт со второго.
- В stderr Codex сыплет нефатальные ошибки (`TokenRefreshFailed`, `failed to record rollout items`, `no such table: jobs`, чужие YAML-варнинги) — на картинку НЕ влияют. Если в конце `Done` и **exit 0** — слайд готов, красный лог игнорируй.
- `exit 10` — единственный реальный стоп: лимит подписки исчерпан → переключайся на Replicate nano-banana-pro (Движок B).

## Движок B — nano-banana-pro (Replicate, FALLBACK + когда нужно много фото)

Как зовём: `replicate.run("google/nano-banana-pro", { input: { prompt, image_input: [...], aspect_ratio } })`. Подробно — `инструменты/replicate.md`. Включается, когда Codex упёрся в лимит подписки (exit 10) ИЛИ нужно много фото-референсов разом.

**Что реально умеет:**
- **Фотореализм и продуктовые фото** — сильна на товаре, сложных композициях, реалистичных сценах.
- **Хорошо держит текст** на картинке (как и gpt-image-2) — Вариант Б работает и здесь.
- **До 14 фото-референсов разом** (`image_input` — массив, потолок 14). Это суммарно на кадр: товар + ракурсы лица + style-anchor. Отсюда «лицо до 14 фото» — можно дать несколько ракурсов одного человека, чтобы намертво зафиксировать личность (тот же кап 14 уже стоит в `04-image-prompt.md`).
- **Соотношения сторон:** `aspect_ratio` — `9:16`, `4:5`, `1:1`, `16:9`, `4:3` и др. Для карусели — вертикаль.
- Цена ~**$0.13/картинка** (1–2K; официально $0.134, проверено WebSearch 2026-06-17; ⚠️ `toolbox replicate.md` с ~$0.04 устарел — то была базовая `nano-banana`, не `pro`). 4K — ~$0.24. Карусель из 5 слайдов ≈ **$0.65**. Для сравнения: Codex/gpt-image-2 по подписке ChatGPT = **$0**. Цены меняются — сверять на день записи.

## Как это читать Дизайнеру (практический слой)

Закладывая промт слайда, помни — движок может ВСЁ это сразу, не проси меньше:

1. **Дизайнерский слайд, не «просто фон».** Заголовок и подпись — внутри кадра, вписаны в сцену (баннер/постер-уровень), а не плоский текст поверх. Кириллица читаемая.
2. **Лицо — что использовать (итог тестов 2026-06-17).** **gpt-image-2 («Image 2», Codex) держит лицо СТАБИЛЬНЕЕ** — для слайдов с лицом бери его. **nano-banana-pro (Google Gemini)** лицо умеет и в последнее время неплохо, но **не так стабильно** — зависит от канала (платформа Vibe через Kie.ai 2K — норм; сторонние Replicate-обёртки — нестабильно, проверено: даже боевой Vibe-промт лицо не удержал). **Для постоянного своего лица на потоке — обучи модель: Flux LoRA (Replicate) или Soul ID (Higgsfield, через платформу), Урок 2** — одиночный фото-референс это не заменяет. Разово через image-модель — Codex + несколько ракурсов; `no beautification` убирать (старит). nano-banana — для товара и сцен без лица (точный 4:5, до 14 фото).
3. **Товар точь-в-точь.** Фото товара референсом + блок identity-lock товара. Форма, логотип, цвет, материалы — без выдумок.
4. **Нужное начертание текста.** Проси стиль шрифта словами: «bold geometric sans», «elegant high-contrast serif», «condensed heavy display». Важная честная граница ⚠️: модель РИСУЕТ текст в нужном *начертании/настроении*, но **не гарантирует конкретный фирменный шрифт пиксель-в-пиксель**. Нужен точный шрифт бренда на всех слайдах одинаково → это код-режим (`05-text-as-code.md`).
5. **Правильный холст.** Вертикаль 1080×1350 (4:5) для ленты, 9:16 для сторис. Не квадрат.
6. **Правка готового кадра** (заменить фон, убрать объект, поправить деталь) — через edit-режим gpt-image-2 API / Flux Kontext, а не перегенерация с нуля.

**Что движок НЕ заменяет.** Если бренду нужен идеально точный шрифт и текст, правимый одной фразой без перерисовки, — это не к image-модели, а к Дизайнеру-код (`05`). Image-модель — про красивый дизайн и сцену; код — про снайперскую точность текста.
