# Карусель из короткого текста — сборщик (методика Vibe)

## Дух скилла (читать первым)

Из одной короткой фразы или куска текста ты собираешь **готовую к публикации карусель**: каждый слайд — отдельная картинка, где **текст нарисован ВНУТРИ кадра** (заголовок + подпись), единый бренд держится через весь набор, лицо и товар остаются точь-в-точь по фото.

**Основной режим — текст рисует image-модель прямо в кадре** (Nano Banana Pro / gpt-image-2; они хорошо держат кириллицу). Так получаются красивые дизайнерские слайды и баннеры, где текст вписан в сцену. Надёжность и единство держатся **системой**: точные строки → стиль-якорь (картинка первого слайда как референс для остальных) → identity-lock лица/товара → единый каркас промтов. Это дословный перенос боевого пайплайна Vibe — он реально работает в продакшене.

**Опциональный режим — текст кодом (вёрстка → PNG)** для случаев, где нужна идеально точная вёрстка и лёгкая правка текста. Он силён в **Claude Code** (тот сам ставит рендер и собирает PNG из HTML/CSS поверх чистого фона); в **Codex** обычно проще встроенный генератор, но и код-вариант можно показать. Подробности — `references/05-text-as-code.md`.

Интерфейс для пользователя простой: он командует по-русски одной фразой. Вся сложность — внутри тебя.

## Три сотрудника карусели (кто внутри делает работу)

Скилл — это бригада из трёх спецов. Ученик командует одной фразой; внутри по конвейеру отрабатывают трое, у каждого своя зона ответственности и своя инструкция.

| # | Сотрудник | За что отвечает | Вход → Выход | Инструкция |
|---|---|---|---|---|
| 1 | **Копирайтер карусели** | раскрутить тему в углы, написать заголовки и подписи слайдов | тема/текст → строки слайдов (headline + body) | `01-text-phase.md` |
| 2 | **Дизайнер на нейросети** (дефолт) | стиль, кино-сцена на слайд, генерация кадра с текстом ВНУТРИ, лицо/товар точь-в-точь | строки + стиль/фото → PNG слайдов | `02-styles` · `03-director` · `04-image-prompt` + паспорт движка `06-модели-возможности` |
| 3 | **Дизайнер на коде** (опция) | тот же кадр, но текст вёрсткой HTML/CSS поверх чистого фона — когда нужна снайперская точность текста и единый фирменный шрифт | строки + фон-картинки → PNG слайдов | `05-text-as-code.md` |

**Конвейер:** Копирайтер отдаёт готовые строки → Дизайнер превращает каждый слайд в картинку. Дизайнер-нейро — дефолт (красивый дизайн, текст в кадре). Дизайнер-код подключается на Claude Code, когда важна точная правка текста и одинаковый шрифт на всех слайдах. Стиль-якорь и identity-lock — общие для обоих дизайнеров.

## Вход (Фаза 0)

Собери (если не задано — бери разумные дефолты, не допрашивай по мелочам):
- **тема или текст** (короткий «3 ошибки новичка в фитнесе» или длинный кусок);
- **сколько слайдов** — дефолт 7, диапазон 3–12;
- **язык** — дефолт русский;
- **стиль** из 5: EXPERT / PHOTOREALISTIC / LIFESTYLE / PRODUCT / CUSTOM (см. `references/02-styles.md`). Это опора, не клетка: подбери ближайший по ТОНУ ниши (бережный wellness/здоровье/услуга → LIFESTYLE; экспертный инфобиз → EXPERT; товар → PRODUCT). Не вгоняй нишу в неподходящий стиль — при нужде бери CUSTOM и опиши свой;
- **фото-референсы**: лицо / товар / стиль (опционально, для identity-lock);
- **бренд-файл** (если есть — цвета/шрифт/тон).

## Пайплайн — 5 фаз (порядок Vibe)

1. **Извлечение углов** — короткий текст → 4–7 разных углов темы (тезис + данные + названия) + идея хука + CTA. Если текста много — берём факты из него и ничего не теряем; если мало — раскрываем тему с разных сторон. Промт: `references/01-text-phase.md` → блок EXTRACT.
   🛑 **СТОП-ГЕЙТ:** покажи пользователю **план** — заголовки/углы слайдов — и спроси «так?» **до** генерации картинок. Картинки стоят денег; план — бесплатно.
   **Сколько углов влезет:** контент-слайдов = **N−2** (минус cover и cta). Тема «5 ошибок / 7 шагов / 10 советов» НЕ равна N слайдов: если пунктов больше, чем N−2 — на стоп-гейте предложи либо увеличить число слайдов, либо объединить/взять сильнейшие пункты (правило из `01-text-phase.md`). Реши это с пользователем ДО картинок, не молча.
2. **Копирайт слайдов** — углы → `headline` + `subtext` по слайдам (cover-hook → content → cta). Один угол на слайд, чередуй углы, объём минимальный. Промт: `references/01-text-phase.md` → блок COPYWRITE.
3. **Визуальный директор** — на каждый слайд **уникальная сцена** (кино-раскадровка, не PowerPoint): своя сцена/ракурс/композиция, без повторов между соседними слайдами. Промт: `references/03-director.md`.
4. **Промт картинки + генерация** — на каждый слайд собери финальный image-промт (сцена + identity-lock + секция TEXT с точными строками) и вызови движок. `references/04-image-prompt.md`. Перед сборкой сверься с паспортом движка `references/06-модели-возможности.md` — заложи весь его потенциал (текст в кадре, лицо до 14 фото, товар, нужный размер, edit), не проси у модели меньше, чем она умеет.
   **Style-anchor + фото — ВСЕГДА оба референса:** cover генерится ПЕРВЫМ; дальше на КАЖДЫЙ слайд 2…N прикладывай ОБА — cover (style-anchor, единый бренд) И фото лица/товара (личность). Не опускай ни один, даже если человек не в кадре на слайде. Детали и команды — `04` раздел 9.
5. **Проверка глазами подписчика** (4 фишки, см. ниже) + сборка папки `slide-01.png … slide-NN.png`.

> **Если image-движок НЕ подключён (частый случай у новичка) — не останавливайся.** Выдай **текстовый результат**: на каждый слайд `headline` + `body` + короткое **описание сцены** (субъект, композиция, свет, layout) с пометкой формата (4:5 / 9:16). Это готовый каркас — пользователь догенерит картинки одной командой, когда подключит движок. Стоп-гейт плана и проверку 4 фишками проходи как обычно. Не выдавай «не могу без движка».

## Генерация картинок (через toolbox)

Зови image-движок через инструменты Виктора: по умолчанию **gpt-image-2** (Codex). ⚠️ **ЛИЦО — что использовать (итог тестов 2026-06-17):** **gpt-image-2 («Image 2», Codex) держит лицо СТАБИЛЬНЕЕ** — бери его для слайдов с лицом. **nano-banana-pro (Google Gemini)** лицо тоже умеет и в последнее время неплохо, но **не так стабильно** (сильно зависит от канала: в платформе Vibe через Kie.ai 2K — норм; через сторонние Replicate-обёртки — нестабильно). **Если нужно своё лицо ПОСТОЯННО / на потоке — обучи модель лица: Flux LoRA (Replicate) или Soul ID (Higgsfield, через платформу) — это правильный путь, Урок 2.** nano-banana удобна для товара и сцен без лица (точный 4:5, до 14 фото). `no beautification` на лице убирать (старит). Формат вертикальный **1080×1350 (4:5)** для ленты, 9:16 для сторис (⚠️ Codex-вертикаль = `1024×1536` = 2:3, ровно 4:5 не даёт — точный `1080×1350` через Replicate `aspect_ratio:"4:5"` или кроп, см. `references/06-модели-возможности.md`). Перед генерацией промт строит skill `визуал-промтер`, если он подключён. Что именно умеет каждый движок и как не растерять его силу в промте — `references/06-модели-возможности.md` (цифры/лимиты сверять на день записи, они меняются).

## Два режима текста на слайде (выбор по среде и задаче)

- **A. Текст в картинке (дефолт) — красивый дизайн.** Image-движок рисует слайд целиком с текстом внутри (Фаза 4 + `04-image-prompt.md`). Работает и в **Codex** (встроенный генератор картинок — ничего не ставить), и в **Claude Code**. Берём по умолчанию — для дизайнерских слайдов, сторис и баннеров.
- **B. Текст кодом (вёрстка → PNG) — надёжная точность.** Фон генерится БЕЗ текста, текст верстается HTML/CSS и рендерится в PNG поверх. Силён в **Claude Code** (сам ставит Playwright/Pillow и рендерит). В **Codex** встроенный генератор обычно проще — код-вариант показываем как «и так можно». Берём, когда нужна точная правка текста и гарантированный единый шрифт. Подробности — `references/05-text-as-code.md`.

**Дефолт — A.** Режим B предлагаем, если пользователь на Claude Code и хочет максимально точный/правимый текст.

## Identity-lock (лицо / товар) — держим точь-в-точь

**Шаг 0 (обязательно): строй lock ОТ АНАЛИЗА фото, не от шаблона.** Посмотри на приложенное фото и опиши реальный объект (лицо: возраст/черты/кожа/причёска как на фото; товар: форма/материалы/лого), вставь описание в промт ПЕРЕД блоком lock. Детали — `04` шаг 6.0.

Блоки lock (из Vibe, `references/04-image-prompt.md`):
- **Лицо:** «match the person's face exactly — eye shape, nose bridge, jawline, lip proportions, hair…». ⚠️ **Итог тестов 2026-06-17:** **gpt-image-2 («Image 2») держит лицо стабильнее**, nano-banana — умеет, но менее стабильно. **Для постоянного своего лица на потоке — Flux LoRA или Soul ID (через платформу), Урок 2.** Если image-моделью — бери Codex + несколько фото-ракурсов; `no beautification` убери (старит). Тот же человек на всех слайдах, меняются поза/ракурс/эмоция/одежда.
- **Товар:** «reproduce the product exactly — shape, proportions, logo placement, label, colorway, materials; invent no new logos, modify no text». Тот же товар, разные ракурсы/сцены по слайдам.

## Единый бренд через весь набор

- **Style-anchor (ВСЕГДА оба якоря):** картинка первого слайда → референс №1 для ВСЕХ следующих, и на каждый слайд вместе с ней — фото лица/товара. Оба прикладываются на каждом слайде, не опускать ни один (модель приоритезирует ранний референс — единый визуальный язык + удержание личности).
- **Consistency-frame:** единая тема фона, привязанная к **литеральному** смыслу темы (про лето — пляж, про бизнес — офис; НЕ «космос/градиент» по умолчанию), единые шрифты и акценты на всех слайдах, включая CTA.

## Методслой — проверка результата (4 фишки автора)

Дизайнером быть не нужно — нужно сменить позицию взгляда. Смотри тремя взглядами: **подписчика** (листает ленту, впервые видит), **клиента** (понял за 2 секунды, чем полезно), **со стороны** (что лишнее). Критерий «получилось» — из позиции получателя, не из своего вкуса. Три проверки:
- первый слайд **останавливает** скролл — или его пролистнут?
- текст **читается** с телефона, кириллица ровная, ничего не наезжает?
- лицо/товар **похожи точь-в-точь**, а не «почти»?

**Готово, если:** папка слайдов в едином фирменном стиле, чистый русский, цепляющий cover → публикуй.

## Границы

Не видео, не монтаж, не оживление кадров. Только карусель / сторис / слайды-картинки.
