# Контент-конвейер — скиллы для недели контента из одного материала

Набор скиллов-ассистентов для код-агента (Claude Code / Codex). Главный из них — **сборщик контента**: вы даёте ему один источник (разбор вашей аудитории, статью, запись эфира или видео) и одну тему — а он за один прогон собирает пакет под площадки: пост, тред, карусель, сценарий Reels, сообщение в Telegram и письмо для рассылки. Плюс картинки в нужных под каждую площадку размерах и папку-план с датами публикации.

## Зачем это нужно

Обычно, чтобы из одного материала сделать контент под все площадки, вы шесть раз переформулируете запрос, руками переносите тексты, отдельно делаете картинки и держите даты в голове. Полдня возни на один день контента — и голос бренда «плывёт» от формата к формату.

Сборщик делает это за **один прогон** и держит ваш голос во всех форматах. Главное: он **ничего не сочиняет с нуля**. Он берёт боль из вашего разбора аудитории, вызывает уже готовый ассистент под каждый формат и раскладывает результат по датам. Один кусок труда → неделя контента.

Идея урока в одной фразе: вы **связываете уже готовые детали**, а не строите контент-фабрику с нуля. Сборщик — тонкий «дирижёр», который вызывает кубики, собранные в прошлых уроках.

## Что внутри

| Скилл | Что делает |
|---|---|
| **content-pipeline** | сборщик-конвейер: один источник + тема → пакет под площадки, раскладка по датам. В нужном порядке вызывает остальные. |
| **carousel-builder** | собирает готовую карусель/сторис — слайды с текстом прямо в кадре, единый бренд, ваше лицо/товар по фото-референсу. |
| **reels-scriptwriter** | сценарий Reels/Shorts: 3 хука на выбор, таймкоды, текст под озвучку. |
| **email-sequence-writer** | серия писем (или одно полезное письмо) из вашего разбора аудитории. |

Пост, тред и сообщение в Telegram сборщик пишет сам — отдельные скиллы под них не нужны.

В папке `skills/content-pipeline/examples/` лежит готовый пример прогона на учебной нише (студия растяжки) — посмотрите, как выглядит результат.

## Безопасность

Это открытый код — можно прочитать каждый файл перед установкой. Эти скиллы не ходят в сеть за вашей спиной и не работают через куки браузера. Главное правило, которое стоит держать всегда: **чужой скилл — это инструкции, которые агент выполняет от вашего имени.** Ставьте скиллы только из источников, которые можете прочитать (как этот), и не ставьте те, что просят доступ к кукам браузера или прячут, куда шлют данные.
